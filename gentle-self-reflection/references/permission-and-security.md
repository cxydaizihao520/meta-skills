# 权限与安全

- **受限 ≠ 不存在**：「权限拒绝/受限」不要呈现为「未找到/不存在/隐藏」。要保留「存在但受限」，否则 agent 可能陷入循环或错误推理。
- **失败即关闭（fail closed）**：后端不可用、权限解析失败或降级时，默认拒绝，不要放行。
- **塑形 + 强制并存**：权限既要生成前的可见性塑形（pre-generation shaping），也要生成后的强制校验（post-generation enforcement），缺一不可。
- **防泄露**：受限输出不得泄露受限字段名、同级实体名、SQL、物理表/列、数据源 ID、主机、用户、堆栈信息。
- **信任稳定标识**：身份与数据范围判断要信任稳定 ID，而不是可变的名称（如角色显示名）。
